## 1. 实现成果

本项目完成了一个**可演示、可答辩、可扩展**的昆明公交旅游路线数据可视化平台，交付物包括：

| 交付类型 | 内容 |
| --- | --- |
| 前端 SPA | Vue 3 + TypeScript + Vite，管理端与游客端分离；线上首页为手绘景点导览页，管理员入口迁移至 `/admin.html` |
| 后端 API | Express REST API，17 个端点（含 CSV 导出、趋势快照、站点详情等），File/Mysql 双数据模式 |
| 静态部署 | 腾讯云 EdgeOne Pages，生产环境通过浏览器端 `mockApi.ts` 替代 Express 后端，支持无服务器运行 |
| 数据库 | MySQL 8 建库建表脚本（7 张核心表）+ 初始化种子数据 + 数据字典 |
| 地图集成 | 高德 JS API 2.0 + AMap.Driving 真实道路路径 + 全线路箭头同步运行 + 3 视角切换 + 速率控制 |
| 采集引擎 | 全局仿真采集器，1500ms 批量写入，驱动全站可视化联动 |
| 仿真模型 | Gravity Model v2（改进重力模型 + 介入机会理论 + 节假日客流 + 32 篇文献标定） |
| 首页体验 | 手绘 scrapbook 风格景点导览页，含 12 个昆明景点、4 条推荐线路、8 道本地美食和隐藏式管理员入口 |
| 工程文档 | 12 份结构化技术文档，完整覆盖需求/设计/接口/数据库/采集/测试/模型/功能说明/总结/配置等 |
| 成果展示 | 答辩 PPT（成果展示.pptx）+ 景点导览 Landing 页 |

## 3. 技术亮点

### 3.1 高德真实道路路径

通过 `AMap.Driving.search()` 为每条线路获取沿真实道路的驾车路径。线路首末站作为起终点，中间各站作为途经点（waypoints），提取返回的 `steps[].path` 拼接为完整道路坐标序列。路径优先级：真实路径 → 手工备用路径 → 直线 polyline。单条线路失败不影响其他线路。

### 3.2 全线路箭头同步运行

10 条线路在地图上同时可见，每条线路上一个 SVG chevron 导航箭头独立移动。统 140ms 动画帧循环，每条线路的 progress 由采集遥测 speed 驱动，箭头方向通过 `setAngle(bearing)` 沿道路实时旋转。点击线路高亮该线路（其余淡化），并展示遥测数据和周边 POI。

### 3.3 秒级动态客流仿真

模型引入毫秒级 `tickJitter` 因子和分钟级种子分量，使得同一秒内连续调用统计接口也返回不同值。观众和评审人员可直接感知"数据在持续变化"。采集因子（`collectionFactor`）连接采集样本与仿真模型，采集数据越多、满载率越高 → 模型输出越大。

### 3.4 ECharts 平滑过渡

数据分析页和采集趋势图的 5 个 ECharts 实例全部采用 `setOption(option, { notMerge: false })` 增量更新，配合 400-600ms `cubicOut` 动画曲线，图表数据变化时平滑过渡，无销毁重建引起的闪烁。

### 3.5 可视化联动

采集器写入样本 → 客流模型重算 → 运营总览数字动画 + 脉冲边框 + 热门排行重排 → 数据采集趋势图增量 → 地图箭头变速 → 遥测面板刷新 → 数据分析图表过渡。全链路数据驱动，无静态写死。

### 3.6 视角切换与智能气泡

3 种地图视角一键切换：自由视角（默认）、跟随车辆（zoom=14 自动追踪）、车内视角（zoom=17 紧贴+旋转匹配朝向）。高亮线路箭头头顶跟随气泡实时显示当前站 → 下一站、客流模拟波动和遥测数据。站点标记点击弹出站点详情窗。

### 3.7 Toast 通知与全局状态栏

右下角 Toast 通知栈实时反馈采集批次、启停和异常事件。顶部全局状态栏显示采集引擎状态指示灯、样本计数、在线线路数和最近刷新时间，模拟智慧城市监控大屏。

### 3.8 双主题系统

暗色/白天一键切换，基于 `data-theme` 属性 + CSS 变量实现。两组完全独立的配色方案，暗色模式亮白文字 + 深蓝黑背景，白天模式深黑文字 + 白色背景。ECharts 图表通过 MutationObserver 监听主题变化自动重绘。

### 3.9 数据图表 Tab

独立的数据可视化面板（ChartPanel.vue），4 个 ECharts 图表 2x2 网格布局：线路客流柱状图、时段客流折线图、热门站点横向柱状图、线路类型占比饼图。零额外地图依赖，与「动态地图」Tab 分工明确。

### 3.10 降级兜底

- 无高德 Key → 内置 SVG 地图 + 备用路径
- AMap.Driving 不可用 → 回退手工备用路径
- 无 MySQL → 自动使用 File 内存模式
- 图片加载失败 → SVG 占位图
- API 请求失败 → 保留上次有效数据

### 3.11 景点导览 Landing 页

独立的手绘风格（Hand-drawn）HTML/CSS 页面。12 个昆明景点卡片排列，每张卡片包含真实图片、Hand-drawn 风格评分、分类标签和"查看公交线路"按钮。点击按钮跳转主平台并携带 `?spot=` URL 参数，主平台自动切换到地图 Tab 并筛选该景点关联线路。采用 Patrick Hand + Kalam 字体，wobbly border 手绘边框，打造轻松有趣的游客导览体验。

### 3.12 CSV 数据导出

GET /api/collection/samples/export 端点将所有采集样本导出为 CSV 文件下载。含 UTF-8 BOM 头适配中文 Excel 直接打开。列包括 id、routeId、routeNumber、speed、passengerCount、loadRate、source、collectedAt。便于答辩现场演示完整的数据采集→存储→导出的数据工程闭环。

### 3.13 趋势快照与站点详情

- 趋势快照 API（GET /api/statistics/trend）：按分钟窗口回溯客流趋势，返回时序数据点 + TOP 线路排行，支持 `?minutes=` 参数自定义回溯窗口。
- 站点详情 API（GET /api/stops/:id）：查询单个站点详细信息，含关联线路列表和停靠顺序。
- 实时时钟：顶部状态栏每秒刷新的数字时钟，模拟智慧城市监控大屏时间显示。

### 3.14 EdgeOne Pages 纯静态部署

V5.1 将原本依赖 Express 的线上运行方式改为纯静态托管。`edgeone.json` 指定构建命令 `cd frontend && npm install && npm run build`，输出目录为 `frontend/dist`。生产环境不配置 `VITE_API_BASE_URL` 时，`frontend/src/api.ts` 自动使用 `frontend/src/mockApi.ts`，把数据查询、客流仿真、管理员登录、采集样本、趋势快照和 CSV 导出都放在浏览器端完成。这样部署到腾讯云 EdgeOne Pages 时不需要额外服务器，也不会暴露后端环境变量。

## 4. 工程规范

| 规范项 | 实践 |
| --- | --- |
| 前后端分离 | Vue 3 SPA + Express REST API，JSON 通信 |
| Repository 模式 | 数据访问层抽象，File/MySQL 双实现 |
| TypeScript | 前端类型定义 + Composition API |
| 统一响应 | `{ data }` / `{ message }` |
| 字段映射 | 前端 camelCase ↔ 数据库 snake_case |
| 代码组织 | 组件化、API 层封装、数据访问层隔离 |
| 文档交付 | 10 份 Markdown 结构文档 + 数据库 DDL/DML + 数据字典 |

## 5. 后续改进方向

- 接入高德 Web Service API 替代 JS API Driving，支持后端预计算 + 缓存路线路径
- 使用 WebSocket 替代轮询，降低网络开销并提升实时性
- 接入真实公交实时到站或车辆定位数据
- 增加用户认证、角色权限和数据导入/导出
- 将 MySQL 模式作为正式部署模式，配置连接池和慢查询监控

## 6. AI 辅助说明

本项目在以下环节使用了 AI 辅助：需求拆解与文档结构化、工程目录设计、Vue 组件代码生成与重构、客流模型公式推演与调参、ECharts 图表配置、高德 Driving API 集成代码、CSS 动画调试。项目中的公交线路名称、站点和景点坐标来源于昆明公开旅游资料的人工整理；客流、拥挤度、准点率等运营指标为基于多因子数学模型的仿真输出，不涉及真实公交运营数据。
