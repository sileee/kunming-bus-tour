## 1. 借鉴来源

本项目在**产品形态和可视化交互**层面参考了以下开源项目：

| 项目 | 作者 | 仓库地址 |
|------|------|----------|
| **bus-vis** | ddiu8081 | [https://github.com/ddiu8081/bus-vis](https://github.com/ddiu8081/bus-vis) |

该项目是一个基于百度地图的公交数据可视化作品，展示了线路网展示、站点信息交互和城市公交数据表达的产品思路。此外，地图轨迹回放中常见的 `Polyline + Moving Marker` 实现模式也是业界广泛使用的技术方案，本项目参考了这一通用技术思路。

## 2. 借鉴内容与独立实现对比

| 维度 | bus-vis (借鉴来源) | 本系统 (独立实现) |
|------|---------------------|---------------------|
| **地图平台** | 百度地图 | 高德 JS API 2.0 |
| **主题定位** | 通用城市公交可视化 | 昆明公交旅游主题 |
| **后端** | 无（纯前端） | Express + MySQL + RESTful API |
| **数据库** | 无 | MySQL，包含线路、站点、样本、景点等多张数据表 |
| **数据采集** | 无 | 仿真采集器持续运行，支持采集因子调节 |
| **客流模型** | 无 | 客流仿真数学模型，动态波动 |
| **后台管理** | 无 | 线路管理、站点管理、数据采集控制面板 |
| **轨迹路线** | 直线/折线连接 | AMap.Driving 真实道路路线，沿路网贴合 |
| **车辆标记** | 简单圆点 | SVG chevron 导航箭头，setAngle 实时转向 |
| **动画方式** | 位置跳变 | 140ms 帧率 setPosition + setAngle 逐帧驱动 |
| **多线路监控** | 单线路切换 | 10 条线路同屏同步运行，高亮/淡化交互 |
| **视角** | 2D 俯视 | 高德 3D 底图（zoom=11，含建筑/地名/POI） |
| **可视化联动** | 静态展示 | 全站数据联动（仪表盘脉冲动画、图表平滑过渡） |
| **地图回退** | 无 Key 则不可用 | 自动回退内置 3D 演示地图 |
| **景点集成** | 无 | 昆明旅游景点坐标、真实图片、详情弹窗 |
| **技术栈** | 前端单页 | Vue 3 + Express + MySQL 前后端分离 |

## 3. 不直接复刻的原因

本项目**没有直接复制或 Fork `ddiu8081/bus-vis` 的代码**，理由如下：

1. **主题不匹配**：原项目为通用城市公交可视化，并非昆明公交旅游主题，不能直接满足课程选题要求。
2. **架构缺失**：原项目为纯前端作品，不含 Express + MySQL 后端、数据库设计和后台管理系统。
3. **课程考核要求**：课程强调源码整理、开发文档和功能完整性，独立搭建更便于在答辩中清晰说明个人实现的内容与工作量。
4. **代码纯净性**：直接克隆后修改容易残留原项目无关代码，影响对工程结构的解释，降低项目可维护性。

**处理方式**：借鉴其产品形态和可视化交互思路，围绕昆明公交旅游主题，独立完成数据模型设计、前后端接口开发、数据库结构搭建和文档体系编写。

## 4. 独立创新点

以下功能为本系统**完全独立设计和实现**，构成原创性贡献：

### 4.1 高德 Driving API 真实道路路线

区别于简单直线连接站点，通过高德驾车路线规划 API 获取真实道路路径，公交车辆沿昆明实际可通行道路行驶，提升模拟真实感。

### 4.2 全线路箭头同屏实时监控

所有线路的车辆 Marker 同屏展示，每辆车附带方向箭头，实时反映行进方向。支持运营人员全景监控多条线路的运行状态。

### 4.3 仿真采集器与客流模型

独立设计的数据采集引擎：
- 约 1.5 秒周期批量生成多条线路样本
- 客流模型参数**动态波动**（速度、车内人数、满载率随机浮动）
- 采集因子可调，控制采集速度和波动幅度
- 跨页面持续运行，切换路由不中断

### 4.4 全站可视化联动

从数据采集到前端展示的完整响应链路：
- 仪表盘客流数字**脉冲动画**
- ECharts 图表**平滑过渡更新**（非直接替换图表实例）
- 首页排行、线路详情、数据分析页同步自动刷新

### 4.5 昆明旅游主题数据集

独立整理的昆明公交旅游线路数据，包括：
- 多条昆明代表性公交旅游线路坐标序列
- 滇池、西山、翠湖等景区真实坐标与图片
- 重点线路轨迹修正（避免穿越湖面/建筑）

### 4.6 自动回退机制

高德 Key 缺失或无效时，系统自动回退为内置 3D 演示地图，所有核心功能保持可用，保障课堂展示和答辩的稳定性。

### 4.7 完整前后端分离架构

独立搭建的 Express + MySQL 后端，包含：
- RESTful API 设计（线路、站点、样本、统计接口）
- MySQL 数据库表结构设计
- 后台管理界面（线路管理、站点管理、采集控制）
