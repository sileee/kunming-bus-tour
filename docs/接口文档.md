## 1. 通用规范

### 1.1 基础地址

```
http://localhost:3000
```

生产环境部署时替换为实际域名或 IP 地址。

### 1.2 统一响应格式

所有接口统一返回 JSON，最外层结构如下：

```json
{
  "code": 200,
  "data": { },
  "message": "success"
}
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `code` | `number` | 业务状态码。200 表示成功；4xx 表示客户端错误；5xx 表示服务端错误 |
| `data` | `object \| array \| null` | 响应载荷。查询接口包裹在 `data` 字段内；`DELETE` 成功时返回空体 |
| `message` | `string` | 错误时的描述信息；成功时通常为 `"success"` 或缺省 |

> **注意**：当前版本的健康检查 `/api/health` 直接返回顶层 JSON，其余业务接口均遵循 `{ data }` 包裹格式。

### 1.3 请求与响应编码

- Content-Type: `application/json; charset=utf-8`
- 请求体使用 JSON 格式
- 时间字段采用 ISO 8601 格式（如 `"2026-05-13T10:30:00.000Z"`）

### 1.4 HTTP 方法约定

| 方法 | 用途 |
| --- | --- |
| `GET` | 查询资源（列表或详情） |
| `POST` | 创建新资源 |
| `PUT` | 全量更新已有资源 |
| `DELETE` | 删除资源 |

## 2. 认证说明

V5.0 起引入 JWT 认证机制，区分管理员与游客权限。

### 2.1 游客端（无需认证）

以下接口为公开接口，游客可直接调用：
- 所有 `GET` 查询接口（线路、站点、景点、统计、采集样本）
- `POST /api/collection/samples`（游客端地图箭头动画所需的采集上报）
- `GET /api/collection/samples/export`

### 2.2 管理员端（需要认证）

管理员接口需要在请求头中携带 JWT Token：

```
Authorization: Bearer <token>
```

需要认证的接口：
- `POST /api/admin/routes`
- `PUT /api/admin/routes/:id`
- `DELETE /api/admin/routes/:id`
- `POST /api/admin/recalculate-heat`

### 2.3 获取 Token

**`POST /api/auth/login`**

请求体：
```json
{ "username": "admin", "password": "admin123" }
```

成功响应：
```json
{ "data": { "token": "eyJhbG...", "username": "admin" } }
```

失败响应（401）：
```json
{ "message": "用户名或密码错误" }
```

### 2.4 验证 Token

**`GET /api/auth/verify`**（需要 Bearer Token）

成功响应：
```json
{ "data": { "valid": true, "username": "admin" } }
```

Token 过期或无效时返回 401。

### 2.5 默认账号

默认管理员账号为 `admin` / `admin123`，可在 `backend/.env` 中通过 `ADMIN_USER` 和 `ADMIN_PASS` 修改。

## 3. 健康检查

### `GET /api/health`

查询服务运行状态及当前数据模式。

**请求示例：**

```
GET http://localhost:3000/api/health
```

**响应示例：**

```json
{
  "status": "ok",
  "name": "昆明公交旅游路线可视化 API",
  "dataMode": "file"
}
```

| 字段 | 说明 |
| --- | --- |
| `status` | 固定返回 `"ok"`，表示服务正常运行 |
| `name` | API 服务名称 |
| `dataMode` | 当前数据模式：`"file"`（JSON 文件存储）或 `"mysql"`（MySQL 数据库） |

## 4. 公交线路

### `GET /api/routes`

查询线路列表。返回每条线路的基本信息、站点、景点、轨迹坐标和动态统计指标。

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `keyword` | `string` | 否 | 线路编号、名称、起点或终点的模糊搜索关键词 |
| `type` | `string` | 否 | 线路类型筛选，如 `"常规公交"`、`"旅游专线"`、`"快速公交"` |
| `spotId` | `number` | 否 | 景点 ID，返回经过该景点的线路 |

**请求示例：**

```
GET http://localhost:3000/api/routes?keyword=翠湖&type=常规公交
```

**响应示例：**

```json
{
  "data": [
    {
      "id": 1,
      "number": "94路",
      "name": "海埂公园 -- 翠湖公园",
      "type": "常规公交",
      "start": "海埂公园",
      "end": "翠湖公园",
      "fare": "2元",
      "operationTime": "06:30 - 22:00",
      "color": "#2563eb",
      "polyline": [[102.6689, 24.9654], [102.6662, 24.9662]],
      "stops": [
        { "id": 1, "name": "海埂公园", "sequence": 1, "lng": 102.6689, "lat": 24.9654, "district": "西山区" }
      ],
      "spots": [
        { "id": 1, "name": "海埂公园", "lng": 102.6689, "lat": 24.9654, "category": "自然景观;公园", "district": "西山区" }
      ],
      "statistics": {
        "passengerFlow": 2847,
        "crowdLevel": "中度拥挤",
        "punctuality": 87.3,
        "heat": 76
      }
    }
  ]
}
```

### `GET /api/routes/:id`

查询单条线路详情。

**路径参数：**

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `number` | 线路 ID |

**请求示例：**

```
GET http://localhost:3000/api/routes/1
```

**响应格式** 与列表接口中单条记录一致，包裹在 `data` 字段内。

**错误响应 (404)：**

```json
{
  "message": "线路不存在"
}
```

## 6. 动态统计

两组统计接口的数据均由后端客流仿真模型根据采集样本实时计算，**非静态写死数据**。

### `GET /api/statistics/overview`

获取首页运营总览概览数据。

```
GET http://localhost:3000/api/statistics/overview
```

**响应示例：**

```json
{
  "data": {
    "routeCount": 12,
    "stopCount": 86,
    "spotCount": 12,
    "simulatedPassengerFlow": 28470,
    "avgPunctuality": 87.3,
    "hotRoutes": [
      { "routeId": 1, "routeNumber": "94路", "passengerFlow": 2847, "heat": 76 }
    ],
    "regionalStopDistribution": {
      "五华区": 24, "盘龙区": 18, "西山区": 22, "官渡区": 16, "呈贡区": 6
    },
    "modelDescription": "客流量由基础需求 * 站点系数 * 景点热度系数 * 线路类型系数 * 早晚高峰系数 * 周末系数 * 随机扰动因子 * 采集样本系数计算得出"
  }
}
```

### `GET /api/statistics/routes`

获取各线路的动态运营指标。

```
GET http://localhost:3000/api/statistics/routes
```

**响应示例：**

```json
{
  "data": [
    {
      "routeId": 1,
      "routeNumber": "94路",
      "passengerFlow": 2847,
      "crowdLevel": "中度拥挤",
      "punctuality": 87.3,
      "heat": 76
    }
  ]
}
```

### `GET /api/statistics/trend`

获取最近 N 分钟的客流量趋势快照序列，用于趋势图回溯展示。

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `minutes` | `number` | 否 | 回溯窗口（分钟），默认 30 |

**请求示例：**

```
GET http://localhost:3000/api/statistics/trend?minutes=30
```

**响应示例：**

```json
{
  "data": {
    "data": [
      { "time": "2026-05-13T10:00:00", "passengerFlow": 28470, "topRoutes": [{"number":"94路","flow":2847}] },
      { "time": "2026-05-13T10:01:00", "passengerFlow": 29120, "topRoutes": [{"number":"44路","flow":2950}] }
    ],
    "count": 30,
    "windowMinutes": 30
  }
}
```

## 7. 数据采集

数据采集是系统的核心模块，模拟公交车 GPS、客流传感器和电子站牌上报数据的过程。采集样本通过客流模型驱动全站统计变化。

> 详细的数据采集方案请参阅 [数据采集方案](./数据采集方案.md)。

### `GET /api/collection/samples`

查询采集样本列表。

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `routeId` | `number` | 否 | 按线路 ID 筛选 |
| `limit` | `number` | 否 | 返回最近 N 条记录，默认返回全部 |

**请求示例：**

```
GET http://localhost:3000/api/collection/samples?routeId=2&limit=20
```

**响应示例：**

```json
{
  "data": [
    {
      "id": 542,
      "routeId": 2,
      "routeNumber": "44路",
      "speed": 28,
      "passengerCount": 51,
      "loadRate": 78,
      "source": "车载客流传感器",
      "collectedAt": "2026-05-13T10:30:15.000Z"
    }
  ]
}
```

### `POST /api/collection/samples`

新增一条采集样本。新增后客流模型自动读取最近样本并重新计算全站统计数据。

```
POST http://localhost:3000/api/collection/samples
```

**请求体：**

```json
{
  "routeId": 2,
  "speed": 28,
  "passengerCount": 51,
  "loadRate": 78,
  "source": "车载客流传感器"
}
```

| 字段 | 类型 | 必填 | 约束 | 说明 |
| --- | --- | --- | --- | --- |
| `routeId` | `number` | 是 | 正整数 | 关联线路 ID |
| `speed` | `number` | 是 | `>= 0` | 瞬时速度 (km/h) |
| `passengerCount` | `number` | 是 | `>= 0` 整数 | 车内实时人数 |
| `loadRate` | `number` | 是 | `0 -- 100` 整数 | 满载率百分数 |
| `source` | `string` | 否 | 最长 64 字符 | 数据来源描述，默认 `"模拟采集终端"` |
| `collectedAt` | `string` | 否 | ISO 8601 | 采集时间，默认服务端当前时间 |

**成功响应 (201 Created)：**

```json
{
  "data": {
    "id": 543,
    "routeId": 2,
    "routeNumber": "44路",
    "speed": 28,
    "passengerCount": 51,
    "loadRate": 78,
    "source": "车载客流传感器",
    "collectedAt": "2026-05-13T10:31:00.000Z"
  }
}
```

**错误响应 (400 Bad Request)：**

```json
{
  "message": "routeId 必须为有效线路 ID"
}
```

### `GET /api/collection/summary`

获取采集概览统计数据。

```
GET http://localhost:3000/api/collection/summary
```

**响应示例：**

```json
{
  "data": {
    "sampleCount": 356,
    "onlineDevices": 12,
    "avgSpeed": 27.4,
    "avgLoadRate": 64.2,
    "latestSamples": [
      { "routeId": 2, "routeNumber": "44路", "speed": 28, "passengerCount": 51, "loadRate": 78 }
    ]
  }
}
```

| 字段 | 说明 |
| --- | --- |
| `sampleCount` | 采集样本总数 |
| `onlineDevices` | 当前在线模拟设备数 |
| `avgSpeed` | 所有样本平均速度 (km/h) |
| `avgLoadRate` | 所有样本平均满载率 (%) |
| `latestSamples` | 最近几条样本摘要 |

### `GET /api/collection/samples/export`

导出全部采集样本为 CSV 文件。

```
GET http://localhost:3000/api/collection/samples/export
```

**响应：** CSV 文件下载，Content-Type: `text/csv; charset=utf-8`，含 BOM 适配中文 Excel。

CSV 列：`id, routeId, routeNumber, speed, passengerCount, loadRate, source, collectedAt`

## 8. 后台管理

后台管理接口用于线路数据的增删改维护操作。

### `POST /api/admin/routes`

新增一条公交线路。

```
POST http://localhost:3000/api/admin/routes
```

**请求体示例：**

```json
{
  "number": "K1路",
  "name": "昆明站 -- 大学城",
  "type": "快速公交",
  "start": "昆明站",
  "end": "大学城",
  "fare": "2元",
  "operationTime": "06:00 - 22:00",
  "color": "#16a34a",
  "polyline": [[102.72, 25.02], [102.75, 25.03]],
  "stops": [
    { "name": "昆明站", "sequence": 1, "lng": 102.72, "lat": 25.02, "district": "官渡区" }
  ]
}
```

**成功响应 (201 Created)：**

```json
{
  "data": {
    "id": 13,
    "number": "K1路",
    "name": "昆明站 -- 大学城",
    "type": "快速公交",
    "start": "昆明站",
    "end": "大学城",
    "fare": "2元",
    "operationTime": "06:00 - 22:00",
    "color": "#16a34a",
    "polyline": [[102.72, 25.02], [102.75, 25.03]],
    "stops": [
      { "id": 100, "name": "昆明站", "sequence": 1, "lng": 102.72, "lat": 25.02, "district": "官渡区" }
    ]
  }
}
```

### `PUT /api/admin/routes/:id`

更新一条已有线路的全部信息。

```
PUT http://localhost:3000/api/admin/routes/1
```

**请求体** 格式同 `POST` 接口。

**成功响应 (200 OK)：** 返回更新后的线路对象，包裹在 `data` 中。

**错误响应 (404)：**

```json
{
  "message": "线路不存在"
}
```

### `DELETE /api/admin/routes/:id`

删除一条线路及其关联的站点数据。

```
DELETE http://localhost:3000/api/admin/routes/1
```

**成功响应 (204 No Content)：** 无响应体。

**错误响应 (404)：**

```json
{
  "message": "线路不存在"
}
```

---

### `POST /api/admin/recalculate-heat`

手动重新计算全部景点热度指数（±30% 随机抖动）。

**请求示例：**

```
POST http://localhost:3000/api/admin/recalculate-heat
```

**成功响应 (200)：**

```json
{
  "data": [
    { "id": 1, "name": "翠湖公园", "heat": 89 },
    { "id": 2, "name": "云南大学", "heat": 92 },
    { "id": 3, "name": "南屏街", "heat": 77 }
  ],
  "message": "景点热度已重新计算"
}
```

**说明：** 仅 File 模式支持。热度值随每次调用变化，驱动客流模型和数据分析图表更新。

## 9. 错误码参考

| HTTP 状态码 | 含义 | 常见场景 |
| --- | --- | --- |
| `200` | OK | 查询/更新成功 |
| `201` | Created | 创建资源成功 |
| `204` | No Content | 删除成功，无响应体 |
| `400` | Bad Request | 请求参数校验失败（如 `routeId` 无效、`loadRate` 超出范围） |
| `404` | Not Found | 请求的资源（线路/站点/景点）不存在 |
| `500` | Internal Server Error | 服务端内部异常（生产环境不返回 `detail`） |

服务端全局错误处理中间件在 `500` 错误时统一返回：

```json
{
  "message": "服务端处理失败",
  "detail": "具体错误信息（仅开发环境返回）"
}
```
