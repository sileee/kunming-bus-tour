## 2. 自动化测试命令

**后端接口烟测：**

```cmd
cmd /c npm run test --prefix backend
```

该命令执行 `backend/tests/api-smoke.test.js`，对所有核心 REST 接口发起 HTTP 请求并校验响应结构与状态码。

**前端类型检查：**

```cmd
cmd /c npx vue-tsc -b
```

该命令对前端工程执行 TypeScript 类型检查，确保所有 `.vue` 和 `.ts` 文件类型安全。

**全量启动（手动验证用）：**

```cmd
cmd /c npm run dev:backend   # 启动后端 http://localhost:3000
cmd /c npm run dev:frontend  # 启动前端 http://localhost:5173
```

## 6. 测试结论

- 后端 17 个 REST 接口全部通过烟测，响应码与数据结构符合预期。
- 前端 TypeScript 类型检查零错误，无类型安全隐患。
- 自动采集器已升级为全局持续仿真：置于 `App.vue` 主应用层运行，跨页面切换不断连；每约 1.5 秒批量生成多条线路运行样本，直接驱动全站可视化联动。
- 数据采集已完整接入客流仿真模型：样本的 `passengerCount`、`loadRate`、`speed` 直接影响 `/api/statistics/overview` 与 `/api/statistics/routes` 的模型计算输出。
- T14--T16 测试用例覆盖实时采集可视化、高德真实道路路径、多线路同步动画三大 V2.0 新特性，全部通过。
- T17--T20 测试用例覆盖景点导览 Landing 页、CSV 导出、趋势快照 API、站点详情 API 四大 V3.0 新特性，全部通过。
- 系统在无高德 Key、无 MySQL 的降级模式下仍可完成全部功能演示，满足课堂展示稳定性要求。
